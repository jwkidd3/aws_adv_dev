Advanced Developing on AWS

A Three-Day, Architecture-First Workshop

Module 1 — Course Overview

Welcome

  • 3 days · 7 lab series · one complete serverless microservices system you migrate yourself
  • Audience: developers who already use AWS and want to build production-grade, cloud-native architectures
  • Lab language: Python 3 with boto3 & AWS SAM CLI
  • ~70% of class time is keyboard time — expect real code, real YAML, real IAM

This is not an introductory course. Familiarity with Python, REST APIs, and basic AWS services is assumed.

The Scenario — Cloud Air

  • Cloud Air is a fictional airline with a monolithic on-premises booking system (Python/Flask on a bare-metal co-located server, one relational DB)
  • The system is brittle, hard to scale, and impossible to deploy without a full-stack outage
  • Over three days you migrate it: strangler-fig decomposition → Lambda microservices → event-driven resilience

Every lab is a chapter in that story. By Day 3 the monolith is retired; Cloud Air runs entirely on serverless AWS services.

  • Day 3 — Resilience & Observability. Polyglot persistence, Step Functions, messaging, event sourcing, CQRS, security, X-Ray. Labs 6–7.

Class runs 09:00–16:00. One-hour lunch, two 15-min breaks. Resource cleanup is part of Lab 7b.

Lab Environment

  • IDE: AWS Cloud9 (m5.large · Amazon Linux 2023 · SSH) in us-east-1
  • User: user1, user2, … — instructor assigns yours at the start of Lab 1a
  • Workspace: ~/environment/aws-adv-dev — cloned from the course repo in Lab 1a
  • Env file: ~/.aws-adv-dev.env exports USER_ID, AWS_REGION, ACCT
  • All resources must be prefixed cloudair-userN-* — shared account, no collision

Cloud9 authenticates via the EC2 instance role — no aws configure, no keys on your laptop.

Ground Rules

  • Do not touch resources outside your cloudair-userN-* prefix
  • Never hard-code credentials — all labs use IAM roles or Secrets Manager
  • Region: us-east-1 exclusively — the IAM policy will deny other regions
  • Bootstrap scripts (bootstrap.sh <labId>) rebuild prerequisites if you fall behind — no need to panic
  • Cleanup is a first-class step: Lab 7b tears everything down with a CloudFormation delete + checklist
